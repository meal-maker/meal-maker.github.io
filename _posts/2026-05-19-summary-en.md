## [Blocking AI bot spam on GitHub with Git's --author flag](https://archestra.ai/blog/only-responsible-ai) ⭐️ 8.0/10

The author describes using Git's --author flag to filter out spammy pull requests from AI bots in their GitHub repository, effectively blocking low-quality automated contributions without relying on platform-level tools. This practical workaround addresses the growing problem of AI-generated spam in open-source projects, which overwhelms maintainers and wastes community resources. It highlights the need for better platform-level anti-spam measures while offering an immediate, low-cost solution. The --author flag in Git's log command allows filtering commits by author name, enabling maintainers to identify and block PRs from specific bot accounts. However, as noted in community comments, this approach has security implications because contributors with merged commits gain elevated privileges in some GitHub workflows.

hackernews · ildari · May 18, 15:24

**Background**: Git's --author flag is used with commands like 'git log --author=<pattern>' to filter commits by author identity. In recent years, open-source maintainers have faced a surge of spam pull requests generated by AI tools, which often contain low-quality or irrelevant code. GitHub currently lacks robust built-in anti-spam features for pull requests, leaving maintainers to devise manual or script-based filters.

Private companies like Vigilant Solutions already aggregate ALPR data from thousands of sources and sell access to law enforcement, as seen in previous ICE contracts.

## [uv 0.11.15 fixes security vulnerabilities, adds TOML v1.1 and Azure signing](https://github.com/astral-sh/uv/releases/tag/0.11.15) ⭐️ 7.0/10

Astral-sh released uv 0.11.15 on May 18, 2026, fixing two security vulnerabilities (a TAR parser differential and an entry point escape issue) and adding backwards compatibility with TOML v1.1 for source distributions as well as support for Azure request signing. These fixes close potentially exploitable gaps in a widely adopted Python package manager, making the supply chain more secure for all users. The TOML v1.1 compatibility ensures uv works with modern configuration files, while Azure signing support streamlines deployment for Microsoft Azure users. The TAR parser differential (CVE-2025-62518) is a low-severity inconsistency in file extraction, while the entry point escape vulnerability could allow scripts to write outside the intended directory. Additional enhancements include stricter validation of wheel filenames, rejection of empty package names, structured error messages for signing failures, and a JSON output option for uv audit.

github · github-actions[bot] · May 18, 19:59

**Background**: uv is a fast Python package and project manager written in Rust, designed as a drop-in replacement for pip and pip-tools. TOML v1.1 added features like inline tables and trailing commas, but not all tools support it yet. Azure request signing lets developers cryptographically sign requests for secure communication with Azure services.
