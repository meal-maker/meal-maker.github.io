## [Intel demonstrates chip accelerating encrypted data computation.](https://spectrum.ieee.org/fhe-intel) ⭐️ 9.0/10

Intel showcased its Heracles chip, which accelerates Fully Homomorphic Encryption computations by up to 5,000 times compared to a top-of-the-line Intel server CPU. This breakthrough makes Fully Homomorphic Encryption practically viable for real-world applications, enabling secure outsourcing of data processing to the cloud without decryption, thus enhancing privacy in areas like healthcare and AI. The chip achieves this performance by decomposing FHE's large numbers into 32-bit data words to reduce latency, though it remains slower than performing the same operations on plaintext data, with exact penalties not specified.

hackernews · sohkamyung · Mar 10, 13:10

**Background**: Fully Homomorphic Encryption (FHE) is a cryptographic method that allows computations to be performed on encrypted data without decryption, producing an encrypted result that matches the plaintext output. This enables privacy-preserving data processing in untrusted environments like cloud servers, but FHE has been computationally slow, necessitating hardware acceleration for practical use.

## [FFmpeg-over-IP tool enables remote GPU-accelerated video processing.](https://github.com/steelbrain/ffmpeg-over-ip) ⭐️ 6.0/10

An open-source tool named FFmpeg-over-IP, developed by 'steelbrain', allows a client machine to offload video processing tasks to a remote server with a GPU, enabling GPU-accelerated video conversion over a network. The system consists of a server component running on the machine with the GPU and a client component that connects to it, requiring only network access. This matters because it decouples video processing hardware requirements from the client environment, enabling GPU-accelerated workflows in cloud-based containers, VMs, or low-power devices that lack dedicated graphics hardware. It represents a practical approach to resource sharing and can improve efficiency for developers and organizations managing distributed media processing pipelines. The client machine does not need a GPU or even a local FFmpeg installation, as it sends commands and data to the remote server which executes them using its local FFmpeg instance. Performance is heavily dependent on network bandwidth and latency, and the project's current implementation requires patching the FFmpeg binary on the server side.

hackernews · steelbrain · Mar 10, 18:26

**Background**: FFmpeg is a widely-used, open-source suite of libraries and programs for handling video, audio, and other multimedia files and streams. GPU acceleration (often via NVIDIA CUDA, Intel Quick Sync Video, or AMD AMF) uses a graphics processor to dramatically speed up video encoding and decoding tasks compared to using only a CPU. The idea of network-accessible processing is a trend in high-performance computing, but FFmpeg-over-IP provides a simpler, more direct client-server model for this specific use case.
