## [用于内存安全 C 编译的 Fil-C 简化模型](https://www.corsix.org/content/simplified-model-of-fil-c) ⭐️ 7.0/10

提出了 Fil-C 的简化模型，这是一种通过胖指针实现内存安全的 C 程序编译技术。该模型旨在使该方法更易于理解和实现。 这很重要，因为它为内存安全提供了一种实用的替代方案，无需将遗留 C 代码用 Rust 等语言重写，从而可能保护大量现有软件基础。它强调了在不放弃成熟 C 生态系统的情况下增强软件安全的持续努力。 Fil-C 使用存储边界等元数据的胖指针，并采用并发垃圾回收来捕获所有内存安全错误，但这种方法可能会引入性能开销，并可能面临与非胖指针 ABI 的兼容性问题。简化模型可能侧重于核心概念，以更清晰地处理这些权衡。

hackernews · aw1621107 · Apr 17, 21:38

**背景**: Fil-C 是 C 和 C++的一种内存安全实现，它使用胖指针和并发垃圾回收来防止缓冲区溢出等错误。胖指针是包含大小或边界等元数据的扩展指针，通过启用运行时检查来增强安全性。Hermetic 构建通过将构建过程与主机系统变化隔离来确保软件构建的可重现性，这对于安全且一致的交付至关重要。
